# "Interpretation Rules.txt"

# Interpretação de gravação automática do Adobe After Effects

# são permitidas linhas em branco, e linhas que começam com "#" são comentários

# formato básico de uma regrá fixa: {requisitos de correspondência} = {definir interpretação}
# formato básico de uma regra flexível: {requisitos de correspondência} ~ {definir interpretação}

# + para cada regra correspondente, a interpretação é definida; curingas ("*" permitidos)
#	em ambos os lados; combinação de curinga corresponde a qualquer coisa, definição de curinga não define nada
# + várias regras podem corresponder, correspondências mais recentes substituem as mais antigas
# + todas as regras são varridas em ordem, uma regra por linha
# + regras são aplicadas após a taxa de proporção de pixels, informações de alfa e rótulo de campo
#	informações lidas de um arquivo

# + regras flexíveis não substituirão valores especificados explicitamente pelo arquivo
#	formatar ou definir por regras anteriores

# largura, altura, taxa de quadros, tipo de arquivo, codec =
#	taxa de proporção de pixels/nome, ordem de campos, taxa de quadros de conformação, interpretação de alfa, código de perfil icc
#	ou descrição do perfil (um ou outro), luz linear

# largura e altura são números inteiros
# a taxa de quadros é um número decimal
# o tipo de arquivo é de quatro caracteres entre aspas (FourCC)
# o codec (aplicável para QT e AVI apenas) também é FourCC

# para descobrir o tipo de arquivo e/ou o FourCC do codec para um tipo específico de gravação,
#	clique com a tecla Option pressionada (Mac) ou clique com a tecla Alt pressionada (Windows) na gravação na janela do projeto;
#	os códigos FourCC aparecerão na última linha de texto ao lado da miniatura

# a taxa de proporção de pixels é uma proporção de números inteiros (ex: 10/11)
#	seguidos opcionalmente por um nome para essa proporção (ex: 10/11/“Minha proporção”)
# No After Effects CS4 (v9), modificamos a taxa de proporção de pixels para vários formatos comuns:
#	D1/DV NTSC de 648/720 (0,9) a 10,0/11,0 (cerca de 0,9091)
#	D1/DV NTSC Widescreen de 6/5 (1,2) a 40/33 (cerca de 1,2121)
#	D1/DV PAL de 768/720 (1,07) a 768/702 (cerca de 1,0940)
#	D1/DV PAL Widescreen de 64/45 (1,42) para 1024,0/702,0 (cerca de 1,4587)

# a ordem de campos é um único caractere, (F=quadro, U=campo superior ou L=campo inferior)
# a taxa de quadros de conformação é um número decimal
# interpretação de alfa: (I=ignorar, S=simples, P=premultiplicação/preto, W=premultiplicação/branco)
#	a interpretação de alfa é ignorada para gravação sem um canal alfa

# o perfi ICC é especificado por código de perfil, que é FourCC, ou por
#	descrição do perfil, que é uma sequência de caracteres entre aspas com no máximo 255 caracteres mais as aspas

# Os códigos de perfil com suporte são:
#	- perfil do espaço de trabalho: "wRGB"
#	- perfil incorporado: "embe"
#	- preservar RGB: "pRGB" (faz exatamente o mesmo que a caixa de seleção Preservar RGB na caixa de diálogo Interpretar gravação)
#	- sRGB: "sRGB"
#	- Adobe RGB: "AS98"
#	- Perfil de filme Kodak 5218: "ginp"
#	- SDTV NTSC: "r6nf" (SMPTE RP 145 primários e ITU-R Rec. 709 TRCs. Preto é codificado em 0 e branco é codificado em 1.)
#	- SDTV NTSC 16-235: "r6ns" (SMPTE RP 145 primários e ITU-R Rec. 709 TRCs. Preto é codificado em 16/255 e branco é codificado em 235/255.)
#	- SDTV PAL: "r6pf" (EBU Tech. 3213 primários e ITU-R Rec. 709 TRCs. Preto é codificado em 0 e branco é codificado em 1.)
#	- SDTV PAL 16-235: "r6ps" (EBU Tech. 3213 primários e ITU-R Rec. 709 TRCs. Preto é codificado em 16/255 e branco é codificado em 235/255.)
#	- HDTV (Rec. 709): "r7hf" (ITU-R Rec. 709 primários e TRCs. Preto é codificado em 0 e branco é codificado em 1.)
#	- HDTV (Rec. 709) 16-235: "r7hs" (ITU-R Rec. 709 primários e TRCs. Preto é codificado em 16/255 e branco é codificado em 235/255.)

# A descrição do perfil pode ser pesquisada na interface do usuário (informações de miniatura da janela do projeto) ou em um
# utilitário de cores, como o ColorSync, ou é definido pelo usuário durante a criação de um perfil personalizado

# luz linear é um inteiro (0=Desativado, 1=Ativado, 2=Ativado para gravação de 32 bpc)
